嵌入式linux-jz2440环境搭建

Posted on 2017-02-21   |   In embedded linux

交叉编译工具必须使用 arm-linux-gcc-3.4.5-glibc-2.3.6

================= 基于nfs服务的烧录 ============================================
// 设置ip地址, 在OpenJTAG> 提示符下
set ipaddr 10.0.0.111
set serverip 10.0.0.138
save

// uboot: 用jlink烧录到norflash中, 就再也不用动它了
nfs 30000000 10.0.0.98:/jz2440/u-boot.bin
nand erase bootloader
nand write.jffs2 30000000 bootloader

// kernel: 在OpenJTAG> 提示符下
nfs 30000000 10.0.0.98:/jz2440/uImage
nand erase kernel
nand write.jffs2 30000000 kernel

// fs: 在OpenJTAG> 提示符下 (实际不用, 用nfs加载fs)
nfs 30000000 10.0.0.98:/jz2440/fs_qtopia.yaffs2
nand erase root
nand write.yaffs 30000000 260000 2f76b40 // 要算文件大小, 很麻烦.

================= uboot / linux 内核打补丁， 编译 ==============================
tar xjvf file.tar.bz2 // 解压
// pwd = uboot
patch -p1 < ../file.patch // 打补丁文件, p1表忽略patch文件内的1层目录
tar cjvf file_new.tar.bz2 dir // 压缩为file_new文件

// pwd = uboot
make clean
make 100ask24x0_config // uboot config文件
make // uboot 编译, 得到bin文件
// 不能用sudo make, 否则就是报错

// pwd = linux-x.x.x
mv 4.3寸LCD_mach-smdk2440.c arch/arm/mach-s3c2440/mach-smdk2440.c
make clean // 清空 (先清空再在SI内查看)
cp config_ok .config // 设置config文件
make uImage // 编译获得内核image

================= 创建文件系统, 挂载NFS文件 ====================================
sudo tar xjf fs_dir.tar.bz2 // 解压缩, 必须加sudo
mkyaffs2image fs_dir image_fs.yaffs // 生成文件系统 image_fs.yaffs

// 手工修改ip地址
ifconfig eth0 10.0.0.111 netmask 255.255.255.0
// 永久修改IP地址
// uboot下使用 set bootargs
// 修改开发板 /etc/init.d/rcS
ifconfig eth0 10.0.0.111

// 开发板手动挂载NFS
mount -t nfs -o nolock,vers=2 10.0.0.98:/jz2440 /mnt/nfs
// 开机自动加载, 修改开发板 /etc/init.d/rcS, 增加两行
mkdir /mnt/nfs
mount -t nfs -o nolock,vers=2 10.0.0.98:/jz2440 /mnt/nfs

// uboot, 从NFS加载文件系统
// * 注意修改/etc/init.d/rcS值, 否则会显示nfs服务器没有响应 ==================== 确认一下 10.0.0.100 对不对, 感觉有点问题
set bootargs noinitrd root=/dev/nfs nfsroot=10.0.0.98:/fs ip=10.0.0.100:10.0.0.98:10.0.0.138:255.255.255.0::eth0:off init=/linuxrc console=ttySAC0
save # 保存修改
reset # 重启. (稍后再重启, 先修改好 filesystem 内的初始化文件)

// 默认启动的文件系统
set bootargs noinitrd root=/dev/mtdblock3 init=/linuxrc console=ttySAC0
save
reset

================== 编译驱动程序 ================================================

  1. pwd=驱动程序目录下, 修改Makefile, 配置内核位置
    KERN_DIR = /mnt/nfs/study/jz2440/kernel/linux-2.6.22.6
  2. pwd = drv_dir 驱动程序目录下
    make // 获得 drv.ko
    arm-linux-gcc -o drvtest drvtest.c // 编译测试程序,
    // -o 表示目标文件, 所以 arm-linux-gcc drvtest.c -o drvtest 一样可以编译成功
  3. 切换到开发板终端 (假设使用nfs加载的系统, 上述编译好后, 直接能在开发板上看到更改)
    insmod drv.ko // 加载驱动模块
    ./drvtest // 执行测试
